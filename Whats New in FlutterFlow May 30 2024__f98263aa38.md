We're excited to bring you new features designed to enhance both the functionality and visual appeal of your projects! From dynamic colors for Pie Charts and Gradients to new merge conflict views, these enhancements are set to elevate your development experience. Dive into the details and discover how these new capabilities can transform your apps!
Upcoming Flutter Version/Package Upgrades ❗
In two weeks, FlutterFlow will be upgrading its Flutter version and several key packages to enhance functionality and performance. For a detailed walkthrough of what to expect and how to prepare, make sure to watch the video below!
Gradient Color by Variable 🌈
This improvement enables dynamic gradient generation based on variable values, allowing for rich, context-sensitive color transitions within your app's design. Ideal for creating engaging visuals that adapt to user interactions, this feature adds a sophisticated touch to gradients!
Pie Chart Color by Variable 🎨
Enhance your data visualizations with the new Pie Chart Color by Variable feature in FlutterFlow. This update allows you to dynamically assign colors to pie chart segments based on variable values, making your charts more intuitive and visually engaging.
Merge Conflicts via "Config Diff" View 🔃
Tackle merge conflicts with clarity using the newly introduced "Config Diff" View in FlutterFlow. This feature simplifies the conflict resolution process by providing a clear and visual comparison of changes, helping you make informed decisions on which versions to accept or modify. Ideal for managing conflicts across app state variables, constants, data types, enums, and API values, the Config Diff View ensures that merging updates becomes a straightforward and error-free process, enhancing your overall development workflow.
Learn more via about the Config Diff view via the Branching documentation here.
Exposed FF-Managed Flutter Tool 💬
FlutterFlow now makes it easier than ever to use the same Flutter environment for both development and debugging. By exposing the FlutterFlow-managed Flutter binary, you can maintain consistency across your development process by using the same Flutter version for editing and running your code. Opt for the "flutterflow run" command in the CLI or set up your IDE to leverage this managed environment directly, ensuring seamless transitions and fewer versioning issues.
